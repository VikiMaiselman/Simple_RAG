class Rectangle:

	# 2. ctor __init__() initializes the object
    def __init__(self, width, height):
 	# we call our accessor methods to set the width and height to Px the user from instantiating width with a negative val

# Python provides a safe and comfortable way of adding getters and setters
# defining them with a @property decorator allows users not to break their code if we introduce getter/setter later in the code (= allows for backwards compatibility)

WHAT IS PYTHON

Programming styles
â€” paradigms for structuring & organizing code

Procedural programming: 
â€” a programming paradigm that focuses on writing procedures or routines that perform operations on data 
â€” uses procedures | subroutines to break down tasks into smaller, manageable pieces
â€” top-down approach 
â€” has IMPERATIVE style of programming (a list of step-by-step instructions to accomplish a task)
â€” procedures manipulate variables (state), they can modify the state directly
â€” characteristics:
	modular (code organized into procedures for reusability)
	sequential execution 
â€” benefits:
	efficiency (instructions can closely map machine instructions)
	clear control flow 
â€” pitfalls:
	mutable state (a source of bugs & difficulty in parallelism)
	limited abstraction 
	code duplication risk increased 
	

Functional programming:
â€” a programming paradigm that focuses on evaluation of mathematical functions
â€” passing data from function to function to function to get a result
â€” functions are treated as data 
â€” functions should be pure (no side effects, no shared state manipulation), avoiding mutable data
â€” has DECLARATIVE style of programming, the emphasis on what should be there result of computation rather than how to compute smth step-by-step
â€” characteristics:
	functions are 1st-class citizens
	pure functions (no side effects, no global data manipulation, same input = same output no matter what)
	use of immutable data 
	emphasis on data transformation 
â€” benefits:
	predictability (no side effects)
	modularity 
â€” pitfalls: 
	performance issues (function invocation overhead)
